## Что было сделано в рамках задания

* **Проектирование базы данных:** Создана модель `Request` для хранения данных клиента (имя, телефон, email, статус, тема, текст и дата создания) с валидацией.
* **Кастомизация панели администратора:** В Django Admin настроен вывод списков, добавлены поисковые индексы по контактам клиентов, фильтры по статусам, а поле даты защищено от редактирования (`readonly_fields`).
* **Разработка публичной части (Django):** Реализован контроллер списков на базе `ListView` с пагинацией и форма добавления заявок на базе `CreateView` + `ModelForm`.
* **Дополнительный функционал CRM:** Реализована страница просмотра заявки с возможностью изменения её статуса ("Новая", "В работе", "Выполнена") через POST-запрос.
* **Frontend-разработка:** Сверстаны интерфейсы с интеграцией Bootstrap 5. Реализован поиск по таблице на стороне клиента на чистом JavaScript (без перезагрузки страницы).
* **Контейнеризация:** Настроена сборка проекта через `Dockerfile` и `docker-compose.yml`.
* **Инфраструктура:** Добавлен конфигурационный файл `.env.example` для хранения секретных ключей (Если они будут).
* **Демонстрационный компонент на React:** Написан SPA-компонент списка заявок, работающий на изолированных имитационных (моковых) данных, с сохранением всей логики интерфейса (цветовые бейджи и моментальный поиск).

## Реализованные страницы и компоненты

### Django-приложение
1. `/requests/` — Главная страница со списком всех существующих заявок, бейджами статусов, кнопкой перехода на создание, строкой JS-поиска и постраничной навигацией (пагинацией).
2. `/requests/create/` — Страница с формой отправки нового обращения. Содержит защиту от CSRF-атак и перенаправляет обратно на список с выводом красивого всплывающего Flash-уведомления.
3. `/requests/<int:pk>/` — Страница обращения. Позволяет менеджеру изучить текст проблемы и переключить статус заявки прямо на ходу.

### React-компонент
* Представлен в виде отдельного React UI-модуля. Отображает mock-данные, полностью копируя структуру и стилистику основной таблицы, демонстрируя работу фронтенд-архитектуры в связке с Bootstrap.
